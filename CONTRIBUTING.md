# Contributing

Thank you for your interest in contributing!

## Developer Certificate of Origin (DCO)

All commits MUST be signed off to certify the Developer Certificate of Origin (DCO).
Use:

```
git commit -s
```

## Branching Strategy

- `main` is the production branch.
- Create feature branches for development (e.g., `feature/your-change`).

## Pull Request Process

1. Fork the repository.
2. Create a feature branch from `main`.
3. Make your changes and commit using `git commit -s`.
4. Push your branch to your fork.
5. Open a Pull Request (PR) against `main`.
